Introduction

The purpose of this lab is to implement two classes that use composition; one class is also a singleton. Both classes may appear again in Lab 06 (graphical user interfaces).

The game Boggle™

Boggle™ is a word game played with 16 dice each having 6 faces labelled with a letter of the English alphabet. One die has a face labelled with the string QU to reflect the fact that Q is usually followed by U in English words.

To start a game of Boggle™, the dice are placed in a dome that covers the game board and the dice are shaken to randomize where each dice appears on the board and to randomize which face is up on each dice. The game board is simply a 4-by-4 grid.

After shaking the dice and making sure that each die has settled into one of the grid positions, the board is placed down and the dome removed. A 3-minute timer is started. The players then search the assortment of letters to find English words made up of three or more letters. Words are formed from letters that adjoin horizontally, vertically, or diagonally with no die being used more than once per word. The image to the left shows an example of how the word QUESTIONS can be formed. Other words that can be formed include EASE, EAST, NOISES, QUEST, and QUESTION.

The longest words that can be formed have 17 letters; the image to the left shows how INCONSEQUENTIALLY can be formed.

Implementing Boggle™ as a Java program

In this lab, students will create 3 essential classes for implementing Boggle™ as a Java program. The completed program has the following appearance when run:

Our Boggle™ app has many features:

  • The player starts the game by pressing the Re-roll button. This causes the dice to be shaken and rolled randomly.
  • It uses buttons to represent the dice; the user clicks on the buttons to form words. When the player clicks on a button, the button background is set to blue and the button is disabled so that the button can only be used once per word. Also, all of the other buttons not adjacent to the last clicked button are disabled; in this way the player is forced to use letters that form a connected path. In the image above, the last button pressed was the E; we can see that only the N and the C buttons are enabled (black text).
  • As the player clicks on the dice buttons, the word being formed is shown in the text field at the bottom middle; in the image above, the user is currently spelling the word DOUSE.
  • The player presses the Submit button when they are finished spelling the word. The word is checked to see if it is spelled correctly and is at least 3 letters long; if both of these are true, then the word is added to the list of correct words. Otherwise, the word is added to the list of incorrect words. In either case, the word is cleared from the text field, and all of the buttons are enabled and set to white so that the player can enter the next word.

You can play the Boggle™ app in the Prism lab using the command: 

java -jar /cse/dept/www/course/2030/labs/lab4/BoggleApp.jar

Ignoring the user interface, we can identify three components needed to implement a Java version of Boggle™:

  1. dice whose faces are decorated with a string
  2. a dictionary to determine if a word has been spelled correctly
  3. a component that models how the game of Boggle™ is played

In this lab, students will create these components by implementing three classes:

  1. Die: A class that represents an n-sided die that can be rolled to display a random face. Each face of the die is decorated with a string. A Die is a composition of a SortedMap that stores which string is associated with each face of the die.
  2. Dictionary: A class that represents a dictionary of English words. A Dictionary is a composition of a SortedSet strings (the words in the dictionary).
  3. Boggle: A class that models how the game of Boggle™ is played. A Boggle object is a composition of 16 Die objects (the 16 dice used in the game) and a composition of a Dictionary.

Hints for implementing the Die class

Rolling a Die to a random face of the die requires generating a random value. java.util.Random can easily be used to generate random integer values within a specified range, but our die faces are labelled with strings. One solution is to assign each face a unique integer value and map that value to the string on the face. Your solution should number the faces of the die 1, 2, 3, ..., n where n is the number of faces on the die, and store the face number and corresponding face string in a SortedMap. For example, consider a 4-sided die with faces labelled "E", "E", "C", "S"; the Die object representing such a die might have the following map:

Key    Value
1 "E"
2 "E"
3 "C"
4 "S"

Die is a composition of a SortedMap; any constructor or method that might leak a reference to the map must make a defensive copy of the map. The Die constructor should create and initialize the map given an array of strings specifying the strings for the faces.

A field for the map is already in the provided Die.java file. You may find that you require additional fields as you implement the various methods; don't forget to initialize these fields to reasonable values in the constructor.

Hints for implementing the Dictionary class

The Dictionary class uses a SortedSet to store the words of the dictionary. We use a set to prevent storing duplicate words. We use a sorted set because certain dictionary operations can be implemented more efficiently if a sorted set is used instead of an unsorted set (although it is not a requirement that students exploit the possible efficiencies). Dictionary is a composition of a SortedSet; any constructor or method that might leak a reference to the set must make a defensive copy of the set.

The dictionary reads in the words of the dictionary from a file named dictionary.txt that has been provided to you. The file contains 370,101 English words in all lower case. A method that reads the file has also been provided for you. Note that the dictionary file contains many acronyms and abbreviations that would not normally be accepted as proper words.

Hints for implementing the Boggle class

The Boggle class is a composition of a Dictionary and a List of 16 Die objects; any constructor or method that might leak a reference to the Dictionary or the List of dice must make a defensive copy of the Dictionary or the List.

The Boggle constructor should create a dictionary and a list to store the dice; it should populate the list with 16 dice. The strings to label the 16 dice are provided to you in the file Boggle.java as a 2-dimensional array named LETTERS.

You can get the 1-dimensional array containing the strings for the faces of the first die by writing: 

String[] faces = Boggle.LETTERS[0];

In the above example, Boggle.LETTERS[0] is the first element of the 2-dimensional array LETTERS; the first element is itself an array of six strings. Similarly, Boggle.LETTERS[1] is the array of six strings for the faces of the second die, and Boggle.LETTERS[2] is the array of six strings for the faces of the third die.

Creating 16 dice can be done by writing a loop: 

for (int i = 0; i < Boggle.NUMBER_OF_DICE; i++) {
    String[] faces = Boggle.LETTERS[i];
    // make a Die using faces here
    // add the die to your list here
}
